How It Works

Three steps from silence to the perfect track.

01 / SEARCH
🔍
Audience Requests
Join with just your name. Search any song from Bollywood to international. Request it live — or if it's already queued, vote it up instead.
02 / VOTE
📊
Crowd Votes
Every song shows live 👍 and 👎 counts. Super votes prioritise requests. The crowd energy meter shows the room's excitement in real time.
03 / PLAY
🎚️
DJ Decides
The DJ sees votes, drags to reorder, smart shuffles by popularity, broadcasts announcements, and marks songs Playing or Played.

Initialization
Get Running in 15 min

No servers. No monthly bills. Just paste two keys and go live.

01
Create Firebase Project
Go to Firebase Console → New Project → Add Web App → copy config. Enable Firestore in test mode. Free Spark plan.
🔗 console.firebase.google.com
02
Allow Your Domain in Firebase
Firebase Console → Authentication → Settings → Authorized Domains → add your Vercel URL. Then in Firestore Rules set: allow read, write: if true;
03
Paste Config into all HTML files
Find the firebaseConfig block in each file and replace with your project's values from Firebase Console → Project Settings.
04
Deploy to Vercel
Push all 4 files to a GitHub repo. Connect on Vercel and deploy. Your site goes live in under 2 minutes.
🔗 vercel.com/new → Import GitHub → Deploy
